# 箱庭TODO

タスク管理 × プロセス監視 × HD-2D ボクセル箱庭のデスクトップアプリ。

「やる気」ではなく「環境」で自分を動かすためのツール。タスクをこなすと報酬として箱庭オブジェクトがもらえて、3D の箱庭が育っていく。達成判定は対象ソフトの起動をプロセス監視で自動検知する。

## 主な機能

### タスク管理
- タスクを作成し、監視対象のプロセスを紐付け
- 現在起動中のプロセス一覧から選択して登録
- 1タスクに複数プロセス紐付け可（いずれか1つの起動で達成）

### プロセス監視
- 30秒間隔のポーリングでプロセスの起動を自動検知
- 「その日にプロセスが起動したか」だけで達成判定（起動時間の閾値なし）
- 起動時間もログに記録して達成履歴で確認可能
- システムトレイ常駐（ウィンドウを閉じても監視は継続）

### 報酬ガーデン
- 達成するとランダムにオブジェクトをドロップ（9種類）
- 8×8 グリッドに自由配置
- オフライン差分計算で時間経過に応じて成長（個体差あり）
- 隣接パターンによる創発エフェクト（17ルール）
- 90日周期でワイプ＆凍結 → ギャラリーに蓄積

### ビジュアル
- Three.js による HD-2D ボクセル描画
- 自前ポストプロセス（Bloom, Tilt-Shift, 色グレーディング, ビネット）
- 設定画面から各エフェクトの ON/OFF・強度調整が可能
- 蛍パーティクル + ポイントライト追従

## 創発ルール

オブジェクトの隣接パターンで自動的に装飾が発生する。詳細は [docs/emergence-rules.md](docs/emergence-rules.md) を参照。

例:
- house×3+隣接 → 町（舗装+ランタン）
- tree×3+隣接 → 森（苔+キノコ）
- shrine+tree → 精霊火（光球浮遊）
- 孤立した大木 → 神木（鳥居+金光粒）

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| フレームワーク | Tauri v2 |
| フロントエンド | React + TypeScript |
| 3D 描画 | Three.js（addons 非依存） |
| バックエンド | Rust |
| データベース | SQLite（rusqlite, bundled） |
| プロセス監視 | sysinfo |

## 開発

```bash
# 依存インストール
npm install

# 開発サーバー起動（Tauri + Vite）
npm run tauri dev

# ビルド
npm run tauri build
```

### 必要環境

- Node.js 18+
- Rust 1.70+
- [Tauri v2 の前提条件](https://v2.tauri.app/start/prerequisites/)

## 対応 OS

- macOS
- Windows

`sysinfo` と Tauri のクロスプラットフォーム対応により、どちらの OS でもビルド・動作する。プロセス名は OS のプロセス一覧から選択するため、OS 間の差異（`.exe` の有無等）は自動的に吸収される。
